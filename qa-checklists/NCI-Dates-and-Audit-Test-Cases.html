NCI Dates and Audit — Test cases (layman)

Indian Standard Time (IST) is used for business dates and audit times. Use Pass/Fail and Notes when you test.

Save as PDF: In Chrome or Edge, press Ctrl+P → choose Save as PDF (or Microsoft Print to PDF) → Save.

Or generate a .pdf file: from the project folder run npm run qa:pdf (creates NCI-Dates-and-Audit-Test-Cases.pdf next to this checklist).

Test 1 — New Case Inward · Entrustment date

In plain English: You should not be able to save a case if the entrustment date is in the future.

What you do: Open New Case Inward. Set Entrustment Date to tomorrow’s date. Try to save.

What you should see: The system should stop you and show a message that the date cannot be in the future.

What you should see: Save should work if all other fields are valid.

Test 10 — Who changed what (on each record)

In plain English: When you create a new record the system should remember who and when.

What you do: Create any record that has Created / Modified columns. After save open the record again or check the list if those columns show.

What you should see: You should see who created it and the date-time of create; modified should match create on first save.

In plain English: When you change a record only “last changed” should move.

What you do: Edit an existing record. Change one field. Save. Open again.

What you should see: Created date and creator should stay the same. Modified date-time should be newer.

Test 12 — Audit log

In plain English: After you save a business record a line should appear in the audit log.

What you do: Create or update a record. Open the Audit Logs screen (if you have access). Find the latest entry for that action.

What you should see: You should see who did it, what action (create/update), and a time that looks like Indian office time—not a confusing foreign time zone.
